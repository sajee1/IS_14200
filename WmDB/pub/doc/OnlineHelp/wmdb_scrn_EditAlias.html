Edit Alias Information Screen
Screen Heading: WmDB > Alias Management > Edit
Use the Edit Alias Information screen to update the connection information for a database alias.
To access this screen, click Alias Management in the WmDB Navigation panel. In the Alias Management Screen, click the Edit button. The server displays the Edit Alias Information screen, which contains the following:
Submit
Applies current settings to the server.
Reset
Clears information that you have entered in the fields and refreshes the screen.
Cancel
Exits this screen without updating the database alias and returns you to the Alias Management Screen.
Alias
Specify the name of the database alias. You cannot change this value.
DB URL
Specify the URL for the database; for example, jdbc:odbc:Support.
DB Username
Specify the user name that the server must supply to log into the database. If a user name and password are not required, leave this field blank.
DB Password
Specify the password that the server must supply to log into the database. If a user name and password are not required, do not click Change Password.
DB Driver
Specify the name of the Java class for the JDBC driver. Type the name of the driver in the box. If the server has any drivers currently loaded, it displays the Loaded Drivers field that lists the drivers currently loaded. You can set the DB Driver field by selecting a driver from the Loaded Drivers list.
Important!  
Be sure the server has access to the Java classes for the driver. Typically, you place the Java classes in the server's classpath.
Minimum Connections
For database connection pooling only. The minimum number of connections to maintain in the database connection pool. The server creates these connections when it first receives a request to connect to the database. If more connections are needed, the server creates additional connections until the value for Maximum Connections, described below, is reached. The default is 0.
Maximum Connections
For database connection pooling only. The maximum number of connections to maintain in the database connection pool. When the number of connection requests reaches this value, the server blocks the requests. The default is 1.
Expiration Time (ms)
For database connection pooling only. The amount of time the server waits before discarding an inactive connection from the database connection pool. The server only discards connections if the pool contains more than the minimum number of connections. Afterwards, if more connections are needed than exist in the pool, the server creates new ones. The default expiration time is 60000 milliseconds.
Loaded Drivers
Specify the names of all JDBC drivers that the server currently has loaded.
